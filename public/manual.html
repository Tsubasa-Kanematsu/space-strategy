概要

rocketDB はロケットの設計データを一元管理するためのWebアプリケーションです。 コンポーネント構成・質量配分・実質量・重心・慣性テンソルを体系的に記録し、 外部 CAD ツールと連携して設計データを自動取り込みすることができます。

主な機能：

  • プロジェクト単位でロケット設計データを管理
  • コンポーネント階層ツリーと数式エンジンによる質量計算
  • 設計版（ロケットDB）のバージョン管理・派生
  • 質量配分値と実質量の比較・差分管理
  • CAD データの自動取り込みと更新履歴管理
  • 重心・慣性テンソルの記録

データ構造

rocketDB のデータは以下の階層構造になっています。

レベル名称説明
1プロジェクトロケットの設計プロジェクト全体。複数のロケットDBを持つ
2ロケットDB設計データの1バージョン。コンポーネントとパラメータを持つ
3コンポーネントロケットを構成する部品・構造体。ツリー形式で階層化

バージョン管理
ロケットDBは派生機能により、既存バージョンをベースに新バージョンを作成できます。 親子関係は設計ツリーで視覚的に確認できます。

プロジェクト一覧

ログイン後の最初の画面です。登録済みのプロジェクトが一覧表示されます。 プロジェクト名または作成者でフィルタリングできます。

プロジェクトを開く

プロジェクト名をクリックすると、そのプロジェクトの設計ツリー（トレーサビリティビュー）に移動します。

プロジェクト作成

  1. プロジェクト一覧右上の 「新規プロジェクト」 ボタンをクリック
  2. プロジェクト名（必須）・作成者名・メモを入力
  3. 「作成」をクリック
注意
プロジェクトを削除すると、配下のすべてのロケットDB・コンポーネント・パラメータも削除されます。

初回DB作成

プロジェクトを新規作成した直後は、ロケットDBがまだ存在しないため設計ツリー画面に 「ロケットDBがありません」と表示されます。同じ画面上の 「最初のDBを作成」 ボタンから、 その場でDB名を入力するだけで最初のロケットDBを作成し、編集画面に遷移できます。

  1. 新規プロジェクト作成後、自動的に設計ツリーに移動
  2. 「最初のDBを作成」ボタンをクリック
  3. DB名 (例: DB v1.0) を入力
  4. そのまま「コンポーネント構成」画面が開く

もちろん、後から設計ツリーの「新規DB」ボタンや派生機能でDBを追加することもできます。

エクスポート / インポート

プロジェクト一覧右上の 「エクスポート」 ボタンで、 すべてのプロジェクトデータを JSON ファイルとしてダウンロードできます。 バックアップや環境移行に使用してください。

設計ツリー（トレーサビリティビュー）

プロジェクトを開くと表示される画面です。 ロケットDBのバージョン関係をSVGツリーで視覚的に確認できます。

アクティブバージョンの管理

各プロジェクトで1つのロケットDBを 「アクティブバージョン」 に設定できます。 設計ツリーでノードを右クリック → 「アクティブバージョンに設定」で変更できます。

コンポーネント構成タブ

ロケットを構成するコンポーネント（部品・構造体）を階層ツリー形式で定義します。 各コンポーネントには質量計算式や変数名を設定します。

コンポーネントは最大6階層まで定義できます。

コンポーネントの列

列名内容
名称 (paramName)表示用の名前。例:「1段エンジン」
変数名 (varName)数式で参照する変数名。英数字とアンダースコアのみ使用可。例: m_eng1
入力タイプ質量の計算方法（下表参照）
値 / 数式数値または数式。リアルタイムで計算結果を表示
ステージall / payload / PBS / stage1〜4

入力タイプ

タイプ説明
fixed固定数値。直接値を入力する
formula数式。パラメータや他コンポーネントの変数名を使用可
aggregate子コンポーネントの合計（自動計算、値の入力不要）

数式エンジン

formula タイプのコンポーネントには、JavaScript 互換の数式を記述できます。 以下の変数・関数が使用可能です。

記述例内容
isp1 * m_prop1 / 9.8パラメータ isp1 とコンポーネント m_prop1 を参照
m_tank * 0.12別コンポーネントの値に係数を掛ける
Math.sqrt(a * a + b * b)JavaScript の数学関数も使用可
依存関係の自動追跡
パラメータやコンポーネントの値を変更すると、 それを参照しているすべての数式が即座に再計算されます。

コンポーネントの追加

  1. ツリー右上の 「ルート追加」 または各行の「+」ボタンで子追加
  2. 名称・変数名・入力タイプを設定
  3. 追加後にツリー上でインライン編集できます

データビュー切替

コンポーネント構成タブのテーブルは、上部のデータビュー切替ボタン群で 表示する列を切り替えられます。同じ階層ツリーを保ったまま、注目したいデータを選んで編集できます。

ビュー表示される列
質量計算質量・実質量・差分・エビデンス
重心座標系 (全機/局所)・cgX / cgY / cgZ (m)
慣性Ixx / Iyy / Izz / Ixy / Ixz / Iyz (kg·m²)
材質材質名・密度・ヤング率
搭載位置mountPos / mountEnd の X/Y/Z (m)

各セルをクリックすると、そのデータ専用の編集モーダル (詳細は「データ変更モーダル」参照) が開き、 値・記入者・エビデンス・添付ドキュメントを記録できます。

質量ビュー

計算質量 (数式エンジンによる) と実質量を並べて表示し、差分を色で可視化します。 親コンポーネントの実質量は、子の実質量を再帰的に積み上げた値が自動表示されます。

実質量セルに 📦 CAD バッジが付いている場合、CADから自動取り込みされた値です。

重心ビュー

各コンポーネントの重心座標 (m) を表示・編集します。 座標系として「全機 (機体原点基準)」または「局所 (搭載位置基準)」を選択でき、 計算エンジンは局所座標を自動的に全機座標に変換して親の重心積み上げを行います。

親 (集計) コンポーネントの重心は、子の質量加重平均で自動計算されます。 値の入力欄は表示されず、計算結果のみが表示されます。

慣性ビュー

各コンポーネントの慣性テンソル (kg·m²) を記録します。 親コンポーネントの慣性テンソルは、平行軸の定理を使って子の慣性テンソルを積み上げた値が表示されます。

材質ビュー

各コンポーネントの材質名・密度 (kg/m³) を記録します。材質プリセットから選ぶと 密度・ヤング率が自動入力されます。

CAD取り込みとの連動
ここで設定した材質密度は、STEPファイル取り込み時 (モードA: 体積×密度) のデフォルト密度として 自動的に使われます。あらかじめ材質を設定しておくと、CAD取り込みがスムーズになります。

搭載位置ビュー

各コンポーネントの搭載位置 (始点と終点) を記録します。 親コンポーネントの搭載位置は、子のバウンディングボックスを集約した範囲が表示されます。

列名内容
mountPosX / mountEndX機軸方向 (X) の始点・終点 (m)
mountPosY / mountEndY横方向 (Y) の始点・終点 (m)
mountPosZ / mountEndZ横方向 (Z) の始点・終点 (m)

親コンポーネントのCG表示モード切替

親 (集計) コンポーネントの重心は計算上は常に全機座標系で算出されますが、 表示モードとして「全機」と「局所」を切り替えられます。

  • 全機モード: 集計CG をそのまま機体原点基準で表示
  • 局所モード: 集計CG から親自身の搭載原点を引いた値を表示 (サブアセンブリ単位の確認に便利)

親コンポーネント行の「全機 / 局所」バッジ (⇄ アイコン付き) をクリックすると切り替わります。 切替は表示のみの変更で、計算ロジックや子の重心積み上げには一切影響しません。

コンポーネントの移動 (再ペアレント)

各行のアクション欄にある 「親を変更」 ボタン ( bi-diagram-3 アイコン) をクリックすると、親選択モーダルが開きます。

  1. 移動したいコンポーネントの「親を変更」ボタンをクリック
  2. 新しい親コンポーネントを一覧から選択 (フルパスで表示)
  3. 「（ルートに移動）」を選ぶと最上位に移動
循環参照の防止
自分自身と子孫コンポーネントは候補リストから自動的に除外されます。 親変更時は階層レベルが再計算され、子孫の階層も再帰的に更新されます。

並び替え

同じ親の中での順序変更は、各行の上下矢印ボタンで行います。

データ変更モーダル

データビューで対象のセルや編集アイコンをクリックすると、データ種別に応じた編集モーダルが開きます。 単一の値だけでなく、関連する複数の値・記入者・エビデンスをまとめて記録できます。

モーダル編集対象
重心 (CG) 編集座標系 (全機/局所) + cgX / cgY / cgZ
慣性テンソル編集Ixx / Iyy / Izz / Ixy / Ixz / Iyz
搭載位置編集X/Y/Z それぞれの始点・終点
フィールド記録個別のフィールド値 (実質量、材質名、材質密度、ヤング率 など)

各モーダルには共通で以下の入力欄があります:

  • 記入者: 入力者の氏名 (変更履歴の changedBy として記録)
  • エビデンス・備考: 算出根拠・参照ドキュメント名など
  • 📎 ドキュメントを添付 (オプション): この変更に紐づくドキュメントを同時に登録

ドキュメント添付

データ変更モーダル下部の 「📎 ドキュメントを添付」 チェックボックスを有効にすると、 図番・タイトル・URLなどの入力欄が展開されます。データ変更と同時にドキュメント情報を コンポーネントの documents に追加できます。

種別に応じた番号ラベル

「種別」を切り替えると、番号フィールドのラベルとプレースホルダーが自動的に変わります:

種別番号ラベル例
図面図番STR-001 / Rev.C
仕様書仕様書番号SPEC-2026-01 / v1.2
レポートレポート番号TR-2026-003 / v1.2
その他文書番号DOC-001 / v1.2
ワークフロー例
「実質量を測定したので試験報告書TR-2026-003を根拠に記録」というケースでは、 実質量編集モーダルで値を入力 → ドキュメント添付をONにして種別「レポート」+ レポート番号を入力 → 記録、の1ステップで値と根拠書類を同時に登録できます。

パラメータタブ

コンポーネントの数式で使用する変数を定義します。 Isp・推力・ΔV など、設計の入力値として扱うデータを登録します。

列名内容
名称表示用の名前。例:「1段 Isp（真空中）」
変数名 (varName)数式で参照する変数名。例: isp1
入力タイプfixed（固定値）/ formula（数式）
値 / 数式数値または数式
活用例
Isp・推進剤密度・タンク肉厚係数などをパラメータとして登録しておくと、 コンポーネントの数式で isp1 のように参照でき、 設計変更時は1か所の変更で全体が再計算されます。

変更履歴と確認ワークフロー

実質量・重心・慣性テンソル・材質・搭載位置などのデータを更新するたびに、 コンポーネントの変更履歴に1エントリが追加されます。 各行のアクション欄にある 時計アイコン (🕐) をクリックすると、 統合変更履歴モーダルが開き、すべてのデータ変更が時系列で表示されます。

ステータス

ステータス意味
入力済・未確認 値は入力されたが、担当者による確認がまだ
CAD取込済・未確認 CADから自動取り込みされたが、まだ確認されていない
✓ 確認済 担当者が確認し承認した値

確認操作

  1. 変更履歴モーダルを開く (時計アイコンをクリック)
  2. 未確認エントリの 「確認」 ボタンをクリック
  3. 確認者名を入力
確認フロー推奨
CAD 自動取り込み後は必ず担当者が数値を確認し「確認済」にすることを推奨します。

親コンポーネントの統合変更履歴

親 (集計) コンポーネントの変更履歴を開くと、子孫コンポーネント全部の履歴も 時系列でマージされて1つのテーブルに表示されます。

  • 第1段全体の履歴を開けば、その下のエンジン・タンク・スカート…全部の変更が一覧できる
  • 「コンポーネント」列が追加で表示され、どの子部品の変更かが識別できる
  • 「確認」操作も親モーダルから子の履歴に対して直接実行可能
活用例
ロケット全体の最近の変更を1画面で確認したいときは、ルートコンポーネントの変更履歴を開いてください。

データファイル取り込み

コンポーネント構成タブの 「CAD取り込み」 ボタンから取り込みモーダルを開きます。 .json / .csv / .stp / .step ファイルを直接アップロードして、 各コンポーネントに質量・重心・慣性テンソル・搭載位置などのデータを反映できます。

取り込んだファイルはセットアップカードとして表示され、いつでも再反映できます。 反映された値には 📦 CAD バッジが表示され、 手入力との区別が一目でわかります。データファイル取り込みも通常の編集と同じく、 変更履歴 (記入者・エビデンス含む) として記録されます。

JSON / CSV ファイル取り込み

CAD ツールから書き出した JSON または CSV を取り込みます。単品・アセンブリの両方に対応。

  1. 「ファイルを追加」ボタンから .json または .csv を選択
  2. ファイル内の各部品名と rocketDB コンポーネントを対応付け (名前一致は自動マッチ)
  3. 「データ反映」ボタンをクリックすると、各コンポーネントに質量・重心・慣性テンソル等が書き込まれる

JSONフォーマットの例

単品:

{ "mass": 45.6, "cg": {"x": 1.2, "y": 0, "z": 0}, "inertia": {...}, "material": {...} }

アセンブリ:

[
  { "name": "Body Tube", "mass": 5.0, "cg": {...} },
  { "name": "Nose Cone", "mass": 1.2, ... }
]

STEP ファイル取り込み

STEP (ISO 10303 / AP203 / AP214 / AP242) ファイルから、 アセンブリ階層と検証プロパティ (体積 / 質量 / 重心 / バウンディングボックス) を抽出します。 単一ファイル (.stp / .step) を直接取り込めます。

  1. 「ファイルを追加」から STEPファイルを選択
  2. セットアップカードに STEP バッジと部品数が表示される
  3. 「詳細設定 / 反映」ボタンを押すと STEP取り込みモーダルが開く
  4. モーダル内でモード・密度・コンポーネント紐付けを設定し、「データ反映へ進む」

取得される情報

情報取得元備考
アセンブリ階層NEXT_ASSEMBLY_USAGE_OCCURRENCE親子関係を完全構築
体積VOLUME_MEASUREm³ に自動換算 (CATIA mm³ 出力に対応)
表面積AREA_MEASURE (wetted area)m² に自動換算
質量MASS_MEASURECATIA で材質割当が必要
重心 (cgX/Y/Z)centre point (CARTESIAN_POINT)絶対座標 (機体原点基準)
搭載位置 (mountPos / mountEnd)bounding box corner point (2点)絶対座標

モードA / モードB

STEP取り込みモーダル上部で、質量算出方法を2つから選べます。

モード条件動作
A: 体積×密度 STEPに体積データがあれば常に利用可 STEPの体積 × 密度 (材質プリセット or 数値) で質量を算出。materialDensity も同時に書き込み
B: 検証プロパティ直接 STEPに質量・重心が含まれている場合のみ有効 STEPの MASS_MEASURE / centre point をそのまま反映

モードBが利用可能な場合はデフォルトでBが選ばれます。利用できない場合は自動的にAになります。

材質との連動

モードAでは、質量計算に使う密度を以下の優先順位で決定します:

  1. 各ノードの個別密度 (材質プリセット選択 or 数値入力)
  2. 紐付け先コンポーネントの materialDensity (材質ビューで設定済みの値)
  3. モーダル上部の共通密度 (デフォルト 2810 = AL7075)
材質を先に設定しておくと便利
コンポーネントに材質名と密度を設定しておけば、STEP取り込み時にデフォルトとして自動使用されます。 コンポーネントの材質を変更すれば取り込み画面の表示も追従し、いつでも上書き可能です。

反映時の更新内容

  • モードA: actualMass + materialDensity + (材質プルダウン選択時) materialName
  • モードB: actualMass + cgX/Y/Z + 慣性テンソル (含まれていれば)
  • 両モード共通: mountPosX/Y/Z + mountEndX/Y/Z (検証プロパティの bbox がある場合)
  • 両モード共通: cgX/Y/Z (centre point がある場合)

反映確認ダイアログ

STEP取り込みモーダルの「データ反映へ進む」ボタンを押すと、 反映情報の入力サブモーダルが開きます。手入力でデータを記録するときと同じく、 記入者とエビデンスを入力してから実際の反映が行われます。

  1. 記入者名を入力 (必須)
  2. エビデンス・備考を入力 (任意。空欄なら自動生成テキストが入る)
  3. 「記録して反映する」をクリック

反映された値はそれぞれ fieldHistory / actualMassHistory に 「入力済 (未確認)」状態で記録されます。担当者が変更履歴モーダルから「確認」操作で承認できます。

搭載位置・重心の反映可否
STEPに bbox や centre point が含まれていれば、モーダルの「搭載位置を反映する」「重心を反映する」 チェックボックスが自動的にONになります。手動でOFFにもできます。

CATIA出力のコツ

CATIA V5 から Batch DXF/IGES/STEP ユーティリティでSTEPを出力する場合、以下の設定で rocketDB が必要とする情報を含めて出力できます。

必須設定

  • ツール → オプション → 互換性 → STEP
  • 「一般」 → ✅ 検証プロパティ (VP) をON
  • 「検証プロパティのパラメータ」 → ✅ 形状検証プロパティ (GVP) を選択
  • 「エクスポート」 → アプリケーションプロトコル: AP242 (推奨)

出力フォーマット

形式圧縮rocketDB対応
STEPなし✅
STEPX (拡張)なし✅
STEPZ (圧縮)あり (.stpz)❌
STEPXZ (拡張+圧縮)あり (.stpxz)❌

質量を出力するには

体積・重心・バウンディングボックスは GVP がONなら自動で出力されますが、 質量は CATIA 上で各部品に 材質を割り当てる必要があります。 メニュー「適用 → 材料の適用」 (Shift+F1) で材質を割り当ててから再エクスポートしてください。

材質割当が難しい場合
CATIA側で材質を割り当てる代わりに、rocketDBのモードA (体積×密度)で 密度を指定して質量を計算する方が手軽な場合もあります。STEPに体積さえあれば、 rocketDB側だけで完結します。

搭載位置の制約 (Master Datum 方式)

パラメトリック設計でマスタージオメトリ (Master Datum Model) パターンを使っている場合、 各部品の組立変換は単位行列になります。CATIAは検証プロパティとして bounding box corner point を絶対座標で出力するため、rocketDB はそこから 正しい mountPos / mountEnd を取得できます。
