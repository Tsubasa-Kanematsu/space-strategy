rocketDB とは

ロケット設計データを一元管理するためのWebツールです。

📊

質量管理

設計値と実測値を並べて差分を即座に可視化

🌲

バージョン管理

設計変更履歴をツリーで追跡

🔗

CAD 連携

外部 CAD のデータを自動取り込み

📝

確認フロー

実質量の更新履歴と担当者確認を記録

データ構造
プロジェクト
　├── ロケットDB（バージョンA）
　│    ├── コンポーネント（ツリー）
　│    └── パラメータ
　└── ロケットDB（バージョンB）
　      └── コンポーネント（引継）
💡 ポイント
1つのプロジェクトに複数バージョンの設計DBを持てます。 バージョン間の親子関係は自動で管理されます。
Section 1

プロジェクト管理

設計プロジェクトの作成・検索・データ保全

プロジェクトの作成と管理

プロジェクトを作成する

  1. サイドバーの「プロジェクト」をクリック
  2. 「新規プロジェクト」ボタンをクリック
  3. プロジェクト名（必須）・作成者名・メモを入力
  4. 「作成」をクリック

フィルター機能

一覧上部の検索ボックスでプロジェクト名または作成者名で絞り込めます。

⚠ 削除について

プロジェクトを削除すると、配下のすべてのロケットDB・コンポーネント・パラメータが完全に削除されます。

バックアップ（エクスポート）

一覧右上の「エクスポート」ボタンで全データを rocketDB-backup-YYYY-MM-DD.json としてダウンロードできます。

🖊 練習

プロジェクトを作成してみよう

  1. プロジェクト名: 「研修用ロケット」
  2. 作成者名: 自分の名前
  3. メモ: 「研修用サンプルデータ」

ロケットDB と設計ツリー

バージョン管理・派生・アクティブ設定

設計ツリー（トレーサビリティビュー）

プロジェクトを開くと設計ツリーが表示されます。DBバージョンの親子関係を視覚的に確認できます。

操作動作
ノードをクリック詳細を右パネルに表示
ノードをダブルクリックそのDBを開く
ドラッグツリーをパン（移動）
ホイールズームイン / アウト
右クリックコンテキストメニュー
🖊 練習

ロケットDBを作成・派生させよう

  1. 「新規ロケットDB」→ 名前「Rev.A」で作成
  2. Rev.A を右クリック → 「派生」→ 名前「Rev.B」
  3. Rev.A を「アクティブバージョンに設定」
DBの作り方 3種類

新規作成

空から定義する

🌿

派生（Fork）

📋

★ アクティブバージョンとは、プロジェクトの現在の正式設計版です。 1プロジェクトに1つだけ設定できます。
Section 3

コンポーネント構成

階層ツリーと数式エンジンで質量を定義する

コンポーネントの定義

「コンポーネント構成」タブでロケットの部品構造を最大6階層のツリーで定義します。

列内容
名称表示名 例: 1段エンジン
変数名数式で参照 例: m_eng1
入力タイプfixed / formula / aggregate
値/数式数値または計算式
ステージall / payload / stage1〜4

入力タイプ

fixed 固定値をそのまま入力
formula 他の変数を使った数式
aggregate 子の合計（自動）
ツリー構成例
ロケット全体 [aggregate]
　├── 1段機体 [aggregate]
　│   ├── エンジン [fixed] 85 kg
　│   ├── タンク [formula]
　│   │   → isp1 * 0.05
　│   └── 推進剤 [fixed] 420 kg
　└── フェアリング [fixed] 30 kg
💡 aggregate の使い方
親コンポーネントを aggregate にすると、子の合計が自動計算されます。 ルートに置くと機体全体の合計質量が得られます。

数式エンジン

入力タイプ formula のコンポーネントに数式を記述できます。
パラメータや他コンポーネントの変数名を使用可能です。

// パラメータ isp1 を使う
isp1 * m_prop1 / 9.8

// 別コンポーネントを参照
m_tank * 0.12

// 数学関数も使える
Math.sqrt(a * a + b * b)
💡 自動再計算
パラメータやコンポーネントの値を変えると、それを参照しているすべての数式が即座に再計算されます。
🖊 練習

コンポーネントツリーを作ろう

  1. ルートに「ロケット全体」(aggregate) を追加
  2. 子に「1段エンジン」(fixed) 85 kg を追加
  3. 子に「推進剤」(fixed) 420 kg を追加
  4. 子に「タンク」(formula) m_prop * 0.08 を追加
  5. 合計質量が自動計算されることを確認
⚠ 変数名のルール
英数字とアンダースコアのみ使用可。先頭は英字。
例: m_eng1 ✅   1st-eng ❌
Section 4

パラメータ

設計変数を定義して数式から参照する

パラメータの使い方

「パラメータ」タブで設計の入力変数を定義します。Isp・推力・ΔVなど、1か所変更すれば全体に反映される値をパラメータとして管理します。

列内容
名称例: 「1段 Isp（真空中）」
変数名数式で参照 例: isp1
入力タイプfixed / formula
値/数式数値 or 計算式

パラメータを使うメリット

  • 設計値を1か所で管理できる
  • 変更時にすべての依存数式が即再計算
  • 感度解析に活用できる
活用例
// パラメータ定義
isp1 = 315 // 1段 Isp [s]
rho_prop = 830 // 推進剤密度 [kg/m³]

// コンポーネント数式で参照
m_tank = m_prop * 0.08
t_burn = m_prop / (thrust / (isp1 * 9.8))
🖊 練習

パラメータを追加しよう

  1. 名称「1段 Isp」、変数名 isp1、値 315 を追加
  2. コンポーネントのタンク質量に m_prop * isp1 / 100000 を設定
  3. isp1 を変えてタンク質量が変わることを確認
Section 5

質量評価

実質量の記録・差分管理・確認ワークフロー

質量評価タブの使い方

「計算質量（設計値）」と「実質量（実測値）」を並べて差分を管理します。

列説明
計算質量数式エンジンによる設計値
実質量手入力または CAD から取り込んだ実測値
差分計算 − 実質量。正＝超過（赤） / 負＝余裕（緑）

実質量を入力する

  1. 末端コンポーネントの実質量セルをクリック
  2. 値 (kg) とエビデンスを入力
  3. 記入者名を入力して「記録」をクリック
💡
親コンポーネントの実質量は子の合計として自動計算されます。
🖊 練習

実質量を記録しよう

  1. 質量評価タブを開く
  2. 「1段エンジン」の実質量セルをクリック
  3. 値: 87.5 kg、エビデンス: 「計測報告書 Rev.1」を入力
  4. 記入者名を入力して記録
  5. 差分の色を確認する
📦 CAD = CAD データから自動取り込みされた値
未確認 = まだ担当者確認が必要な値

更新履歴と確認ワークフロー

実質量を更新するたびに履歴エントリが自動記録されます。
時計アイコン 🕐 をクリックして履歴を確認できます。

ステータスの種類

入力済・未確認 値は記録されたが確認待ち
CAD取込済・未確認 CAD自動取り込み後、確認待ち
✓ 確認済 担当者が確認・承認した値

確認操作の手順

  1. 🕐 アイコンをクリックして
    更新履歴モーダルを開く
  2. 最新エントリの「確認する」ボタンをクリック
  3. 確認者名を入力して「確認する」
📋 推奨フロー
CAD取り込み後は必ず担当者が数値を確認してから「確認済」にする運用を推奨します。
🖊 練習

確認フローを実行しよう

  1. 🕐 アイコンをクリックして履歴を開く
  2. 「確認する」→ 確認者名を入力
  3. ステータスが「確認済」になることを確認
Section 6

重心・慣性テンソル

コンポーネントの物理特性データを記録する

重心・慣性テンソルタブ

重心データ（CGタブ）

各コンポーネントの重心座標をロケット基準座標系 (m) で記録します。

列内容
X (m)機軸方向（ノーズ基準）
Y (m)機軸直交方向
Z (m)機軸直交方向
エビデンス算出根拠・CADファイル名

慣性テンソル（慣性テンソルタブ）

各コンポーネントの慣性モーメント・慣性乗積 (kg·m²) を記録します。

列内容
Ixx / Iyy / Izz慣性モーメント (kg·m²)
Ixy / Ixz / Iyz慣性乗積 (kg·m²)
💡 インライン編集
各セルをクリックして直接編集できます。
Enter で確定、Esc でキャンセルです。
📦 CAD連携
パラメトリック CAD モデルの「CAD更新 & 質量反映」を実行すると、 重心・慣性テンソルも自動で書き込まれます。 手入力の手間が大幅に削減されます。
🖊 練習

重心座標を入力しよう

  1. 「重心」タブを開く
  2. 「1段エンジン」の X 座標セルをクリック
  3. 値 4.250 を入力して確定
  4. エビデンスに「CAD Rev.3 参照」を入力
Section 7

CAD データ連携

JSON取り込みとパラメトリック CAD モデルの活用

CAD データ取り込み — JSON 方式

質量評価タブの 「CADデータ取り込み」 ボタンから開きます。

JSON ファイルの取り込み手順

  1. 「CADデータ取り込み」モーダルを開く
  2. JSON ファイルをドラッグ&ドロップ or クリックして選択
    CAD から出力した .json ファイル
  3. 適用先を設定
    単品 → コンポーネント1つを選択
    アセンブリ → 部品ごとに対応付け
  4. 「JSON 取り込む」をクリック
取り込まれるデータ
  • 質量 (kg)
  • 重心 X / Y / Z (m)
  • 慣性テンソル Ixx〜Iyz (kg·m²)
💡 バッジ表示
取り込み後、実質量セルに 📦 CAD バッジが表示されます。 手入力値との区別がひと目でわかります。
📝 更新履歴
CAD取り込みも手入力と同様に更新履歴が記録されます。 確認フローで「確認済」にすることを忘れずに。

CAD データ取り込み — パラメトリックモデル

STEP / JSON / CSV ファイルからアセンブリ構造と質量・重心・慣性プロパティを取り込み、rocketDB のコンポーネントに反映します。

ワークフロー

  1. 「CADファイルを追加」で STEP / JSON / CSV をアップロード
  2. 「バインド設定」でパラメータと CAD alias を紐づけ
    例: rocketDB の D_body → CAD の Spreadsheet alias D_body
  3. コンポーネントと CAD オブジェクトを対応付け
  4. 「CAD更新 & 質量反映」ボタンをクリック
対応データ形式
STEP .stp / .step (ISO 10303)
JSON アセンブリ階層・質量プロパティ
CSV フラットなパラメータ表
💡 複数モデル登録
複数のデータファイルを登録して、それぞれ独立したバインド設定を持てます。 「バインド設定」ボタンで各モデルの設定をモーダルで確認できます。

バインド設定の詳細

パラメータバインド

rocketDB の値を CAD の Spreadsheet alias に送ります。

設定項目説明
CAD aliasCAD の Spreadsheet alias 名
ソースrocketDB のどの値を送るか（パラメータ・定数・コンポーネント など）
倍率単位変換係数（m→mm なら 1000）
送信値実際に送られる値のプレビュー（緑=OK/赤=未設定）

コンポーネントバインド

CAD のオブジェクトと rocketDB コンポーネントを対応付けます。

設定項目説明
CAD オブジェクト名CAD のオブジェクト名（自動検出可）
rocketDB コンポーネント書き戻し先のコンポーネント
密度 kg/m³質量計算用の密度（空欄=材質密度を使用）
💡 複数オブジェクトの合算
同じコンポーネントに複数の CAD オブジェクトをバインドすると、 質量を合算・重心を質量加重平均・慣性テンソルを加算します。 タンク本体＋バルブ等の構成部品の合算に便利です。
💡 送信値プレビュー
バインド設定モーダルの「送信値」列で、 実際に CAD に送られる値を事前確認できます。
緑=解決済 / 赤=未設定
🖊 確認

バインド設定を確認しよう

  1. セットアップカードの「バインド設定」をクリック
  2. 送信値列が緑になっているか確認
  3. 赤の場合はソース設定を修正する
